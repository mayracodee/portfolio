Portfólio pessoal desenvolvido para apresentar minha trajetória, meus conhecimentos, minha evolução e meus projetos como Desenvolvedora Front-End.

Este projeto representa uma etapa importante da minha transição para a área de tecnologia. Durante o desenvolvimento, coloquei em prática conhecimentos de HTML, CSS e JavaScript, trabalhando não apenas a parte visual, mas também organização de código, responsividade e interações com o DOM.

## ✨ Sobre o projeto

O objetivo foi criar um portfólio com identidade própria, capaz de apresentar quem eu sou, minha jornada de aprendizado e a forma como venho evoluindo na área de desenvolvimento Front-End.

A interface foi construída do zero e adaptada para diferentes tamanhos de tela, com atenção à organização das seções, experiência de navegação e consistência visual.

O projeto possui seções para:

- apresentação profissional;
- sobre mim;
- habilidades técnicas;
- jornada de desenvolvimento;
- diferenciais profissionais;
- formas de contato;
- download do currículo.

## 📱 Responsividade

O projeto foi adaptado e testado em diferentes larguras de tela.

Foram trabalhados breakpoints específicos para:

- desktop;
- tablets maiores;
- tablets menores;
- dispositivos móveis.

Durante essa etapa, foram realizados ajustes em:

- grids;
- alinhamentos;
- espaçamentos;
- tamanhos de elementos;
- menu de navegação;
- imagens;
- botões;
- organização dos cards.

Também foram identificados e corrigidos problemas reais de overflow causados por elementos com largura fixa.

Os ajustes responsivos foram centralizados no arquivo `responsive.css`, facilitando a manutenção e a visualização do comportamento da interface em diferentes tamanhos de tela.

## 📂 Organização do projeto

A estrutura do projeto foi organizada separando os arquivos por responsabilidade, facilitando a manutenção e a localização de cada recurso.

```text
portfolio/
├── assets/
│   ├── docs/
│   └── images/
├── css/
│   ├── reset.css
│   ├── responsive.css
│   ├── style.css
│   └── variables.css
├── js/
│   └── main.js
├── index.html
└── README.md
```

### Responsabilidade dos arquivos

- `assets/docs/` — armazena documentos utilizados no projeto, como o currículo em PDF;
- `assets/images/` — reúne as imagens utilizadas na interface;
- `css/reset.css` — remove e normaliza estilos padrão aplicados pelos navegadores;
- `css/responsive.css` — concentra os ajustes de responsividade e breakpoints;
- `css/style.css` — reúne os estilos principais e componentes visuais do projeto;
- `css/variables.css` — centraliza variáveis reutilizáveis, como cores da identidade visual;
- `js/main.js` — contém as interações e comportamentos implementados com JavaScript;
- `index.html` — estrutura principal da aplicação;
- `README.md` — documentação do projeto.

## 💡 Principais aprendizados

Mais do que finalizar uma interface, este projeto me ajudou a compreender melhor como uma página é estruturada e como diferentes elementos se relacionam dentro do layout.

Durante o desenvolvimento, evoluí principalmente em:

- compreensão da estrutura HTML;
- relação entre elementos pai e filho;
- organização de layouts;
- uso de Flexbox e CSS Grid;
- criação de responsividade;
- identificação de problemas visuais;
- investigação da causa de overflow;
- adaptação de componentes para diferentes telas;
- organização dos estilos por responsabilidade;
- uso de JavaScript para controlar estados da interface.

Um dos maiores aprendizados foi entender que responsividade não significa apenas diminuir elementos, mas analisar como cada componente deve se comportar em diferentes larguras.

Também aprendi a investigar a causa de problemas antes de aplicar correções. Durante a adaptação para dispositivos móveis, por exemplo, identifiquei um elemento com largura fixa maior que a tela, que causava overflow e desalinhamento na interface.
